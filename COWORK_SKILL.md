# 案件応募スキル（ルーター）- Cowork自走版

## フェーズ構成

案件応募フローは4フェーズに分かれている。Coworkのスケジュールタスクで各フェーズを時間差で起動する。

| フェーズ | 指示文の例 | 参照ファイル | ブラウザ |
|---------|-----------|------------|---------|
| Phase 1: スクレイピング | 「スクレイピングを実行してください。スキルシフトで。」 | `SCRAPE_SKILL.md` | 要 |
| Phase 2: 応募文作成 | 「応募文を作成してください。スキルシフトで。」 | `DRAFT_SKILL.md` | 不要 |
| Phase 3: フォーム送信 | 「応募を実行してください。スキルシフトで。」 | `APPLY_SKILL.md` | 要 |
| Phase 4: スプシ同期 | 「スプシ同期してください。」 | `python sync_sheets.py` | 不要 |

---

## ルーティングルール

### 「スクレイピングを実行」「候補を収集」と言われた場合
→ `SCRAPE_SKILL.md` の手順に従う

### 「応募文を作成」「ドラフトを作成」と言われた場合
→ `DRAFT_SKILL.md` の手順に従う

### 「応募を実行」「フォーム送信」と言われた場合
→ `APPLY_SKILL.md` の手順に従う

### 「スプシ同期」「sync」と言われた場合
→ `python sync_sheets.py` を実行する

## Cowork スケジュール設定

各フェーズを時間差で起動する。前フェーズのバトンファイルがなければ自動停止する。
**各サイト max_applications_per_run: 5件**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【チイキズカン】
08:00  スクレイピングを実行してください。チイキズカンで。
08:05  応募文を作成してください。チイキズカンで。
08:10  応募を実行してください。チイキズカンで。

【スキルシフト】
08:30  スクレイピングを実行してください。スキルシフトで。
08:35  応募文を作成してください。スキルシフトで。
08:40  応募を実行してください。スキルシフトで。

【ハイプロ】
09:00  スクレイピングを実行してください。ハイプロで。
09:05  応募文を作成してください。ハイプロで。
09:10  応募を実行してください。ハイプロで。

【スプシ同期（全サイト分まとめて）】
09:30  スプシ同期してください。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**時間設計の根拠：**
- Phase 1（SCRAPE）: 一覧を読むだけ → 約2分
- Phase 2（DRAFT）: ブラウザ不要、テキスト生成のみ → 約1分
- Phase 3（APPLY）: フォーム送信5件 → 約15〜20分
- Phase 4（SYNC）: HTTP POST → 約1分
- サイト間は20分間隔（Phase 3の完了を待つ余裕）

## ファイルの流れ（バトン方式）

```
Phase 1: SCRAPE
  └→ inbox/{site}_{timestamp}_candidates.jsonl   ← バトン(1)

Phase 2: DRAFT（バトン(1)がなければ停止）
  └→ inbox/{site}_{timestamp}_drafts.jsonl        ← バトン(2)

Phase 3: APPLY（バトン(2)がなければ停止）
  ├→ inbox/{site}_{timestamp}_applied.jsonl       ← バトン(3)
  └→ applied_urls.txt に追記（即時・1件ごと）

Phase 4: SYNC（バトン(3)がなければ「処理対象なし」で終了）
  ├→ applied.jsonl をスプシに反映
  └→ processed/ に移動
```

---

## 重複防止

- applied_urls.txt に応募済みURLを蓄積
- Phase 1: 一覧取得時にapplied_urls.txtと照合 → 応募済みはスキップ
- Phase 3: 送信前にもapplied_urls.txt + applied.jsonlをダブルチェック
- Phase 3: 1件送信するごとに即座にapplied_urls.txtに追記 → クラッシュしても安全

---

## 注意事項

- Coworkはブラウザ操作にComputer Useを使用する
- 各サイトには事前にブラウザでログインしておくこと
- PCの電源が入っている必要がある（ローカル実行のため）
- candidates.jsonl と drafts.jsonl は sync_sheets.py の処理対象外（action が "candidate" / "draft" のためスキップ）
