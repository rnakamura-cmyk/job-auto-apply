# 案件応募自動化システム

このシステムは、副業サイトで案件を自動検索・応募し、結果をGoogleスプレッドシートに記録するツールです。

## Claude Codeへの指示

**このファイルを読んだら、まず以下のメッセージをユーザーに送ってください。他のステップは何もせず、返答を待ってください。**

---

```
案件応募自動化システムが見つかりました。

このシステムをセットアップしますか？
（認証設定・プロフィール入力・ファイル登録まで一緒にやります）

y で進める / n でキャンセル
```

---

**「n」が返ってきたら「了解です。必要なときはいつでも声をかけてください。」と伝えて終了してください。**

---

**「y」が返ってきたら、以下のフェーズを順番に実行してください。**

---

## フェーズ1：技術セットアップ

### 1-1. Pythonパッケージのインストール

```bash
pip install google-auth google-auth-oauthlib google-api-python-client
```

実行して完了を確認する。

---

### 1-2. credentials.json の確認

`credentials.json` が存在するか確認する。

**存在しない場合**、ユーザーにこう伝える：

```
Google Cloudの認証ファイルが必要です。以下の手順で取得してください（10分ほど）：

1. https://console.cloud.google.com/ を開く
2. 新しいプロジェクトを作成（名前は何でもOK）
3. 「APIとサービス」→「ライブラリ」→「Google Sheets API」を有効化
4. 「APIとサービス」→「認証情報」→「サービスアカウントを作成」
   - 名前: job-automation（何でもOK）
   - ロール: 編集者
5. 作成したサービスアカウントをクリック →「キー」タブ →「鍵を追加」→「JSON」
6. ダウンロードしたファイルを credentials.json にリネームして、このフォルダに置く

完了したら「置きました」と教えてください。
```

返答を待ち、「置きました」が来たら次へ進む。

---

### 1-3. スプレッドシートの設定

ユーザーにこう伝える：

```
Googleドライブで新しいスプレッドシートを作成してください。

作成したら：
1. URLの https://docs.google.com/spreadsheets/d/【ここ】/edit の部分をコピー
2. スプレッドシートの「共有」ボタンを押し、以下のメールアドレスを「編集者」として追加：
```

`credentials.json` の `client_email` を読み取り、メールアドレスをユーザーに表示する。

続けて：

```
3. スプレッドシートIDをここに貼り付けてください
```

スプレッドシートIDを受け取ったら `config.json` の `spreadsheet_id` を更新する。

---

### 1-4. 動作確認

```bash
python3 sync_sheets.py
```

「inbox/ に処理対象ファイルなし」と表示されれば成功。ユーザーに「技術セットアップ完了です！」と伝える。

---

## フェーズ2：プロフィール設定

以下の質問を1つずつ聞いてください。まとめて聞かず、返答を受けてから次の質問へ。

**Q1**
```
あなたのお名前・ビジネスネームを教えてください。
（例：むらりょう / 中村崚）
```

**Q2**
```
一言で表すと、どんな仕事が得意ですか？
（例：GASとAIで業務を自動化するフリーランス）
```

**Q3**
```
得意な領域・スキルを教えてください。箇条書きでOKです。
（例：GAS・スプレッドシート自動化、n8n/Difyでのワークフロー構築、など）
```

**Q4**
```
実績を1〜3つ教えてください。数字があると◎です。
（例：受注管理を自動化して月20時間の削減を実現、など）
```

**Q5**
```
応募したくない仕事の条件はありますか？
（例：常駐・対面必須・デザイン系・語学系、など）
```

**Q6**
```
希望の単価・稼働時間はありますか？（任意）
（例：月5万円〜、週5〜10時間、など）
```

全回答を受け取ったら `my_profile.md` に以下の形式で書き出す：

```markdown
# プロフィール

## 基本情報
- 名前: {Q1の回答}
- キャッチコピー: {Q2の回答}

## 得意領域
{Q3の回答を箇条書きで}

## 実績ハイライト
{Q4の回答を箇条書きで}

## 応募NG条件
{Q5の回答を箇条書きで}

## 希望条件
{Q6の回答}
```

書き出し後、「プロフィールを保存しました！」と伝える。

## フェーズ3：添付ファイル登録

ユーザーにこう伝える：

```
履歴書・ポートフォリオなど、応募時に添付したいファイルはありますか？

あれば、このチャットにドラッグ＆ドロップして送ってください。
documents/ フォルダに保存します。

なければ「なし」と教えてください。
```

ファイルが送られてきたら `documents/` フォルダに保存する。
複数ファイルも受け付ける。
「なし」が返ってきたらスキップ。

## フェーズ4：応募サイト選択

ユーザーにこう伝える：

```
応募するサイトを選んでください（複数可）：

① クラウドワークス
② スキルシフト
③ ハイプロ Direct
④ ふるさと兼業
⑤ オタノミ
⑥ チイキズカン
⑦ LIFULL LOCAL MATCH
⑧ ロッツフル

番号で教えてください（例：①② / 全部 / 後で決める）
```

選択されたサイトに対応する `sites/{サイト名}/config.json` を確認し、必要であれば設定変更を案内する。

---

## フェーズ5：テスト実行

ユーザーにこう伝える：

```
セットアップ完了です！

テスト実行しますか？
（選択したサイトにアクセスして案件を検索します）

y でテスト実行 / n でスキップ
```

「y」が返ってきたら、選択されたサイトに対して `COWORK_SKILL.md` の手順を実行する。

## 日常の使い方

### Coworkへの指示（案件応募）

```
# クラウドワークスで応募
案件応募を実行してください。クラウドワークスで、[カテゴリ名]で。

# スキルシフトで応募
案件応募を実行してください。スキルシフトで、[カテゴリ名]で。

# 複数サイト同時
案件応募を実行してください。クラウドワークスとスキルシフト両方で。
```

### スプレッドシートへの同期

```bash
python3 sync_sheets.py
```

または5分ごとに自動実行されるスケジュールタスク「crowdworks-auto-sync」が自動処理します。

## ファイル構成

```
automation_bridge/
├── CLAUDE.md              ← このファイル（セットアップ案内）
├── COWORK_SKILL.md        ← Coworkへの指示ルーター
├── my_profile.md          ← あなたの強み・実績（セットアップで自動生成）
├── my_profile.example.md  ← プロフィールの記入例
├── config.json            ← スプレッドシートID・カラム定義
├── sync_sheets.py         ← Google Sheets同期スクリプト
├── requirements.txt       ← 必要なPythonパッケージ
├── documents/             ← 履歴書・ポートフォリオ等の添付ファイル
├── sites/
│   ├── crowdworks/
│   ├── skillshift/
│   ├── hipro/
│   ├── furusatokengyo/
│   ├── otanomi/
│   ├── chiikizukan/
│   ├── lifull/
│   └── lotsful/
├── inbox/                 ← Coworkが結果を書き出す場所
├── processed/             ← 処理済みファイル
├── error/                 ← エラーファイル
└── logs/                  ← 実行ログ
```

---

## 対応サイトを追加したいとき

```
sites/{サイト名}/config.json  ← 検索条件
sites/{サイト名}/SKILL.md     ← フォームの埋め方
```
の2ファイルを作るだけ。`sync_sheets.py` は変更不要。
